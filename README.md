## ✨ Особенности

- 📂 **Продвинутый файловый менеджер**: Навигация, контекстное меню (создание, копирование, вырезание, вставка, переименование, удаление), поддержка специальных путей (например, `::{20D04FE0...}` для "Этот компьютер").
- 📊 **Системная панель (Dashboard)**: Отображение информации о дисках, свободном месте и базовых характеристиках системы.
- 📜 **Редактор реестра**: Безопасный и структурированный просмотр ключей и параметров реестра Windows.
- 🧹 **Очистка системы**: Автоматическая очистка временных файлов, кэша, папок Prefetch и Recent.
- 📝 **Журнал действий (History)**: Встроенное логирование всех действий пользователя через `HistoryService`.
- 🎨 **Современный UI**: Темная тема, адаптивная верстка и иконки в стиле Fluent Design.

## 🛠️ Технологический стек

- **Язык**: C# (.NET 8)

- **Архитектура**: MVVM (Model-View-ViewModel)

## 🚀 Начало работы

### Требования
- Windows 10 / 11
- [.NET 8.0 SDK](https://dotnet.microsoft.com/download/dotnet/8.0) или новее
- Visual Studio 2022 (с установленной рабочей нагрузкой ".NET Desktop Development")

### Установка и запуск
1. Перейдите в раздел релизов репозитория и скачайте исполняемый файл:  
   🔗 [WREX Releases](https://github.com/FrazzieYT/WREX/releases)
2. *(Для разработчиков)*: Клонируйте репозиторий, откройте файл решения `WREX.sln` в Visual Studio, дождитесь загрузки пакетов NuGet и нажмите `F5` для запуска.

### ⚠️ Примечание о среде восстановления (WinRE / Windows PE)
В текущем виде приложение **не запускается** в среде восстановления Windows (WinRE) из-за отсутствия предустановленного `.NET 8 Desktop Runtime` и ограничений графической подсистемы WinPE.  
**План развития:** Для полноценной работы в WinRE планируется создание кастомного образа Windows PE с последующим внедрением (.NET Runtime) с помощью утилиты `DISM`, а также адаптация логики работы с оффлайн-реестром и файловой системой.

## 📂 Структура проекта

Проект строго следует паттерну MVVM для обеспечения чистоты кода и тестируемости:

```text
WREX/
├── Models/             # Классы данных (DriveInfoItem, RegistryTreeNode и т.д.)
├── Views/              # XAML-разметка пользовательского интерфейса
├── ViewModels/         # Логика представления, команды и INotifyPropertyChanged
├── Services/           # Бизнес-логика (FileSystemService, CleanupService, HistoryService)
├── Converters/         # ValueConverters для XAML (статусы, цвета)
└── App.xaml            # Точка входа и глобальные ресурсы
